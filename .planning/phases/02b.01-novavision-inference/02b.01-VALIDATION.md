# Phase 02b.01 — Validation Strategy

## Purpose

Phase 02b.01, mevcut Phase 2B NovaVision entegrasyonunu gerçek CLI davranışına uydurur.

Doğrulanan gerçek CLI komutları:

```text
novavision install
novavision start
novavision stop
```

Geçersiz komut:

```text
novavision deploy
```

Bu yüzden ana validation hedefi: live path içinde `novavision deploy` çağrısı kalmaması ve app_id tabanlı local start/inference kontratının test edilmesidir.

## Test Infrastructure

| Property | Value |
|----------|-------|
| Framework | pytest + FastAPI TestClient |
| Quick mock command | `cd server && uv run pytest ../tests/phase02b -q` |
| Focused command | `cd server && uv run pytest ../tests/phase02b ../tests/test_novavision_live.py -q` |
| Full backend gate | `cd server && uv run ruff check . && uv run ruff format --check . && uv run mypy . && uv run pytest ../tests/ -q` |
| Live marker | `@pytest.mark.live` |

## Required Automated Checks

From `server/`:

```bash
uv run ruff check .
```

## Manual Live Check

Prerequisites:

```env
NOVAVISION_MOCK=false
NOVAVISION_TOKEN=...
NOVAVISION_INFERENCE_URL=http://localhost:8501
NOVAVISION_DEFAULT_APP_ID=<APP_ID>
NOVAVISION_TEST_APP_ID=<APP_ID>
NOVAVISION_TEST_IMAGE_PATH=/absolute/path/to/image.jpg
```

Commands:

```bash
novavision start server
novavision start app --id <APP_ID>
curl http://localhost:8501/health
```

